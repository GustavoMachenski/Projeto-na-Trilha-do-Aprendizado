SOFTWARES DE VERSIONAMENTO

Os softwares de versionamneto servem para controlar as versões de um documento, seja esse docuemtno de uma pessoa individual ou um grupo de peossas.

As versões de um documento dizem respeito as alterações feitas nele, ou seja, o documento original é a versão 1, caso ocorra alguma mudança nesse documento ela sera a versão 2.

REPOSITÓRIOS

Temos 2 repositórios, são eles o:

  • Repositório local

    • É um repositorio que está em sua maquina.

  • Repositório remoto

    • É um repositorio externo a sua maquina, como um servidor.

Os dois repositorios são controlados pelo software de versionamento facilitando a organização.

TIPOS DE SOFTWARES DE CONTROLE

CENTRALIZADO

Tem um repositório central ligado em um servidor com controle de versões. Todos os menbros que utilizam este documento trabalham nesse servidor.

Uma desvantagem é que esse sistema é totalmente dependente do servidor.

DISTRIBUÍDO

Ele tem um repositório local com controle de versões e faz atualizações em um repositório central.

Uma vantagem é que não depende do servidor.

GIT

O git é um software livre de versionamento criado por LINUS TORVALDS, nele cada diretório é um repositorio com histórico completo e não depende de rede ou servidor central.

GITHUB

O github é uma plataforma que guarda os códigos-fonte versionados pelo Git.

DESKTOP GITHUB

Ele é reponsalvel por fazer a integração do Git e o GitHub

Comandos GIT

  • Commit

    • Serve para tornar permanente todas as alterações em um repositório.

  • Push

    • Envia alterações para o servidor no repositorio remoto.

  • Fetch

    • Verefica se o repositório local e remoto estão sincronizados.

  • Pull

    • Sincorniza o repositório local e o remoto atualizando as versões.

Esses comando serão executados via GitHub Desktop!
